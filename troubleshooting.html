Troubleshooting

Debug commands, common issues, and pitfalls to avoid.

Debug Commands

Check Hammerspoon console:

Open Hammerspoon → Console (Cmd+;) or check the log file at:

~/Library/Logs/Hammerspoon/

List audio devices (AVFoundation):

ffmpeg -f avfoundation -list_devices true -i "" 2>&1 | grep -A 20 "audio devices"

Use this to find the correct device ID for batch recording mode.

List audio devices (SDL2):

whisper-stream -m ~/whisper-models/ggml-base.en.bin

Run this and check the startup output for the SDL2 device list. Used for streaming mode.

Test microphone recording:

ffmpeg -f avfoundation -i :3 -ar 16000 -ac 1 -t 5 /tmp/test.wav && afplay /tmp/test.wav

Replace :3 with your audio device ID. Records 5 seconds and plays it back.

Test batch transcription:

whisper-cli -m ~/whisper-models/ggml-medium.en.bin -f /tmp/test.wav

Transcribes the test recording. Verifies model loading and transcription pipeline.

Test streaming:

whisper-stream -m ~/whisper-models/ggml-base.en.bin --step 500 --length 5000 --keep 200 -kc -nf -fa -c 1

Starts streaming transcription in terminal. Press Ctrl+C to stop.

Check if whisper-stream is running:

pgrep -f whisper-stream

Returns process ID if running, empty if not.

Kill stuck whisper-stream:

pkill -f whisper-stream

Terminates all whisper-stream processes. Use if overlay becomes unresponsive.

Reload Hammerspoon:

killall Hammerspoon; sleep 1; open -a Hammerspoon

Clean restart after config changes. Never use hs.reload() or URL schemes.

Check Hammerspoon version:

/Applications/Hammerspoon.app/Contents/Frameworks/hs/hs -c "hs.processInfo.version"

Verifies Hammerspoon installation and version.

Common Issues

No text appears after recording (batch mode)

Problem: You press the hotkey, record, but no text gets typed.

Solutions:

  • Check if whisper-cli is installed: which whisper-cli
  • Verify model path in whisper-stt.conf — the file must exist
  • Test recording manually with the ffmpeg command above
  • Check Hammerspoon console for error messages
  • Verify accessibility permissions: System Settings → Privacy & Security → Accessibility → Hammerspoon (enabled)

Overlay doesn't appear (streaming mode)

Problem: Pressing Ctrl+Shift+Cmd+9 does nothing.

Solutions:

  • Check Hammerspoon console for hotkey registration confirmation
  • Verify Logi Options+ is remapping mouse button correctly
  • Ensure all required Lua modules are installed: whisper_streaming.lua, whisper_overlay.lua, whisper_mouse.lua in ~/.hammerspoon/
  • Check that init.lua loads all modules with require()
  • Try triggering with keyboard shortcut directly to isolate mouse button issue

Mouse buttons don't trigger streaming

Problem: MX Anywhere 3S side buttons don't start/stop streaming.

Root cause: MX Anywhere 3S side buttons don't send CGEvents — this is by design from Logitech.

Solution:

  • You MUST use Logi Options+ to remap side buttons to keyboard shortcuts
  • Forward button → Ctrl+Shift+Cmd+9 (start streaming)
  • Back button → Ctrl+Shift+Cmd+0 (stop streaming)
  • Test with keyboard shortcut directly first to isolate the issue
  • Verify the remap is working by checking what shortcut fires in Hammerspoon console

Wrong microphone detected

Problem: System is transcribing from laptop mic instead of USB/Bluetooth mic.

Solutions:

  • List devices: ffmpeg -f avfoundation -list_devices true -i ""
  • The self-healing check should auto-correct, but you can manually edit WHISPER_AUDIO_DEVICE in whisper-stt.conf
  • Important: SDL2 device IDs (streaming mode) differ from AVFoundation device IDs (batch mode)
  • Your "AirPods Pro" might be ID 3 in AVFoundation but ID 1 in SDL2

Hallucination text appearing on overlay

Problem: Phantom text like "[BLANK_AUDIO]", "(upbeat music)", or "Thank you." appears during silence.

Root cause: Whisper models generate hallucinations during silent or noisy input.

Solution:

  • Check whisper_streaming.lua for the hallucination filter patterns
  • Common filtered patterns: [BLANK_AUDIO], (upbeat music), Thank you., standalone you
  • If new patterns appear repeatedly, add them to the filter in whisper_streaming.lua
  • The filter is case-insensitive and runs on every transcription chunk

Copilot corrections not working

Problem: Streaming overlay shows raw text without grammar/punctuation fixes.

Solutions:

  • Check if auth file exists: cat ~/.config/careless-whisper/auth.json
  • Token format should be ghu_* (GitHub user token)
  • Use menubar → Sign In to Copilot to authenticate via OAuth
  • Without Copilot, streaming still works — raw text is displayed without correction
  • Check Hammerspoon console for API errors (rate limits, network issues)

F5 triggers macOS Dictation instead of Whisper

Problem: Pressing F5 starts Apple's built-in dictation, not Whisper.

Solution:

  • Go to System Settings → Keyboard → Dictation
  • Change or disable the Dictation shortcut (default is F5 double-press)
  • Don't use F5 for Whisper hotkeys unless Apple Dictation is fully disabled

Common Pitfalls

  1. F5 conflicts with macOS Dictation — Don't use F5 unless Dictation is disabled in System Settings → Keyboard → Dictation.
  2. Cmd+7/8/9 conflict — Many apps (browsers, terminals) use these for tab switching. That's why this setup uses Ctrl+Shift+Cmd+9/0 instead.
  3. PATH not available in Hammerspoon — Hammerspoon doesn't inherit your shell PATH. Always use full paths: /opt/homebrew/bin/ffmpeg, /opt/homebrew/bin/whisper-cli, /opt/homebrew/bin/whisper-stream.
  4. Background processes die — Use nohup and redirect output to persist processes after the launching script exits.
  5. whisper-cli syntax — The file argument is positional (not a -f flag): whisper-cli -m model.bin file.wav, not whisper-cli -m model.bin -f file.wav.
  6. Hammerspoon reload — Config changes don't auto-apply. Always use killall Hammerspoon; sleep 1; open -a Hammerspoon. Never use hs.reload() or URL schemes — they cause state corruption.
  7. hs.execute vs hs.task — Use hs.task.new() for scripts with arguments or long-running processes, not hs.execute() which blocks the main thread.
  8. MX Anywhere 3S CGEvents — Side buttons don't generate CGEvents that Hammerspoon can intercept. Logi Options+ remap to keyboard shortcuts is required.
  9. SDL2 vs AVFoundation IDs — The same microphone has DIFFERENT IDs in each API. SDL2 device #1 may be AVFoundation device #3. Always check both separately.
  10. Whisper hallucinations — The model generates phantom text during silence or noise. Always have filtering enabled in streaming mode.
  11. nohup for background processes — Without nohup, processes launched from Hammerspoon scripts may terminate unexpectedly when the parent script ends.

Configuration Reference

The main config file is ~/whisper-mac/upstream/Careless-Whisper/whisper-stt.conf. Key variables:

Variable Default Description
WHISPER_AUDIO_DEVICE 3 AVFoundation device ID for batch recording
WHISPER_LANGUAGE en Language for transcription
WHISPER_MODEL ~/whisper-models/ggml-medium.en.bin Model path for batch mode
WHISPER_CORRECTIONS (in-repo TSV) Path to transcription_corrections.tsv
Note: Streaming mode uses its own settings hardcoded in whisper_streaming.lua: SDL2 device, base.en model, step/length/keep parameters. These can be changed by editing the Lua file directly.
