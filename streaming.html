⭐ Live Streaming Mode

Real-time speech-to-text with visual overlay and AI grammar correction — the star feature of this project.

How It Works

Press the mouse Forward button (or Ctrl+Shift+Cmd+9) to start streaming. The whisper-stream binary begins capturing audio via SDL2 from your USB microphone. Every 500ms, partial transcriptions appear on a dark overlay at the bottom of your screen.

When a complete sentence is detected (period, question mark, exclamation), it's sent to GitHub Copilot for grammar and spelling correction. The corrected text flashes green briefly, then fades to white.

Press the mouse Back button (or Ctrl+Shift+Cmd+0) to stop — any remaining gray (partial) text is automatically promoted to white (confirmed), the overlay briefly shows the final result, then the text is copied to clipboard and pasted into the active app. The audio is also saved as a WAV file for later re-transcription if needed.

The Overlay

A dark semi-transparent panel appears fixed at the bottom of your screen (80% width, 120px height). Visual elements include:

  • White text — confirmed transcription (final words)
  • Gray text with blinking ▌ cursor — partial transcription (still evolving)
  • Green flash — text just corrected by Copilot (fades to white over 2 seconds)
  • Red ● STREAMING badge — top-right corner indicator

Built with hs.webview (Hammerspoon's native WebKit panel).

Hallucination Filtering

Whisper generates phantom text during silence — common patterns like [BLANK_AUDIO], (upbeat music), Thank you., Thanks for watching., Bye., Bye bye., (soft music), [MUSIC], [silence], (gentle music), Thank you for watching., Cheers., [Music], you.

These are detected by pattern matching in whisper_streaming.lua and silently discarded. Only real speech reaches the overlay. The filter uses both exact matches and pattern-based detection (brackets, parentheses).

Copilot Grammar Correction

Trigger: sentence boundary detected (., ?, !)

Endpoint: POST https://api.githubcopilot.com/chat/completions

Auth: Bearer token from ~/.config/careless-whisper/auth.json (ghu_* format)

Required headers: Editor-Version (vscode/1.120.0), Editor-Plugin-Version (copilot-chat/0.35.0), Copilot-Integration-Id (vscode-chat)

Prompt: lightweight "fix grammar/spelling, return only the corrected text" with temperature 0.1

Model: claude-sonnet-4.6 (configurable in whisper_streaming.lua)

Uses hs.http.asyncPost() for native async HTTP (no process spawning). Graceful fallback: if no auth file exists, raw text is kept as-is.

whisper-stream Output Format

Uses ANSI escape codes: \033[2K\r for partial text overwrites, \n for confirmed text. The streaming engine in whisper_streaming.lua parses these codes to distinguish partial vs confirmed text.

With --step 500 --length 5000, text is confirmed every ~4.5 seconds.

Audio Archiving

Streaming mode now saves audio to ~/Documents/claude_projects/whisper/upstream/Careless-Whisper/recordings/ as timestamped WAV files (e.g. 20260409153835.wav). This uses whisper-stream's native --save-audio flag — the process working directory is set to the recordings folder so files land in the right place.

Both batch mode and streaming mode save to the same recordings/ directory. Streaming mode automatically retranscribes on stop using the larger medium.en model, applies dictionary corrections from transcription_corrections.tsv, and wraps the output with LLM-aware metadata. The manual re-transcribe (Ctrl+Shift+Cmd+R or middle mouse button) is still available for batch recordings or to re-run retranscription.
