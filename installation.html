Installation Guide

Complete step-by-step setup for AI agent replication. Follow every step in order.

Prerequisites

  • macOS on Apple Silicon (Metal-accelerated whisper.cpp)
  • Homebrew package manager
  • Git, Python 3.10+
  • A USB microphone (or built-in MacBook mic)
  • A mouse with programmable side buttons (optional, for streaming-mode triggers)
  • GitHub Copilot subscription (optional, for grammar correction in streaming mode)

Step 1: Install Homebrew Packages

Install whisper.cpp, audio recording, and OCR (used for project attribution from screenshots):

brew install whisper-cpp ffmpeg tesseract
brew install --cask hammerspoon
  • This installs: whisper-cli (batch transcription), whisper-stream (live streaming), whisper-server, ffmpeg (audio recording), and tesseract (OCR for bin/extract_project_from_screenshot.py)
  • Verify: which whisper-cli → /opt/homebrew/bin/whisper-cli
  • Verify: which whisper-stream → /opt/homebrew/bin/whisper-stream
  • Verify: which tesseract → /opt/homebrew/bin/tesseract

Step 2: Clone the Repository

Clone whisper-mac — Careless-Whisper is vendored under upstream/, so no separate clone is needed:

git clone https://github.com/dawidbednarczyk/whisper-mac.git ~/whisper-mac
cd ~/whisper-mac

Step 3: Install Python Dependencies

The transcription log, dashboard renderer, and worker need a Python venv:

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
Re-activate the venv in every new shell that runs bin/transcription_worker.py, bin/backfill.py, or bin/_html_render.py. Hammerspoon-launched workers should use the absolute interpreter path ~/whisper-mac/.venv/bin/python3.

Step 4: Download Whisper Models

The 6-stage pipeline needs three GGML models (base.en, medium.en, large-v3). Use the helper script — it downloads, verifies SHA-256 against models.sha256, and is idempotent:

./bin/install_models.sh
  • Models land in ~/whisper-models/ (read by whisper-stt.conf and hammerspoon/whisper_streaming.lua).
  • Sizes: ggml-base.en.bin ~150 MB, ggml-medium.en.bin ~1.5 GB, ggml-large-v3.bin ~3.1 GB.
  • Re-verify any time: cd ~/whisper-models && shasum -a 256 -c ~/whisper-mac/models.sha256
⚠️ Required for the log dashboard: All three models must be present. bin/transcription_worker.py runs every recording through 6 stages (base/medium/large-v3 × raw/corrected); a missing model leaves rows stuck in queued forever.

Step 5: Wire Up Hammerspoon

Hammerspoon provides hotkeys, mouse triggers, the streaming overlay, and the self-healing mic check. Sync the Lua files into ~/.hammerspoon/ with the helper:

./bin/sync_hammerspoon.sh
  • Open Hammerspoon from Applications.
  • Grant Accessibility permissions: System Settings → Privacy & Security → Accessibility → enable Hammerspoon.
  • Grant Microphone and Input Monitoring permissions when prompted.
  • Reload Hammerspoon after every config edit:
killall Hammerspoon; sleep 1; open -a Hammerspoon
⚠️ Warning: NEVER use hs.reload() or the URL scheme to reload Hammerspoon. Always use killall + open. Some config changes don't auto-apply.

Step 6: Configure Batch Mode

Configure the batch transcription settings:

  • Edit ~/whisper-mac/upstream/Careless-Whisper/whisper-stt.conf:
    • WHISPER_AUDIO_DEVICE=3 (AVFoundation device ID for your mic — run ffmpeg -f avfoundation -list_devices true -i "" to find yours)
    • WHISPER_LANGUAGE=en
    • WHISPER_MODEL=~/whisper-models/ggml-medium.en.bin
  • Test recording: ffmpeg -f avfoundation -i :3 -ar 16000 -ac 1 -t 5 /tmp/test.wav
  • Test transcription: whisper-cli -m ~/whisper-models/ggml-medium.en.bin -f /tmp/test.wav

Step 7: Configure Mouse Buttons (Optional)

If your mouse has programmable side buttons, map them to the streaming hotkeys via the vendor's utility (e.g. Logi Options+ for Logitech, Steermouse for generic mice):

  • Forward button → Ctrl+Shift+Cmd+9 (start streaming)
  • Back button → Ctrl+Shift+Cmd+0 (stop streaming)
Why the indirection: Many mouse side buttons don't generate standard macOS CGEvents, so Hammerspoon cannot detect them directly. The vendor utility remaps them to keyboard shortcuts that Hammerspoon CAN detect. See Controls Reference for full details.

Step 8: Configure Copilot Auth (Optional)

Enable grammar correction in streaming mode with GitHub Copilot:

mkdir -p ~/.config/careless-whisper
echo '{"token":"ghu_YOUR_COPILOT_TOKEN"}' > ~/.config/careless-whisper/auth.json

Or use the menubar: Hammerspoon menubar → Sign In to Copilot

Optional Feature: Copilot auth is optional. Without it, streaming mode works perfectly — you just don't get automatic grammar correction.

Step 9: Verify the Transcription Log

The log captures every recording across 6 pipeline stages (base.en + medium.en + large-v3, raw and corrected) into ~/.whisper_log/records.json and renders a dashboard at ~/.whisper_log/transcriptions.html.

After making your first batch recording, kick the log so the dashboard renders, then open it (re-activate the venv first if you opened a new shell):

source .venv/bin/activate
python3 bin/backfill.py 1
open ~/.whisper_log/transcriptions.html

You should see one row with status badges for each stage. Stages that are still queued show queued; once the worker picks them up they flip to running and finally done.

Reference: See Transcription Log for the schema, project attribution, and troubleshooting.

Step 10: Verify Everything

Run through this checklist to confirm your installation is working:

  1. Hammerspoon shows in menubar (○ icon)
  2. Press Ctrl+Cmd+W — icon changes to ● with timer
  3. Speak something, press Ctrl+Cmd+Q — text appears in active app
  4. Press mouse Forward button — overlay appears at bottom of screen with red STREAMING badge
  5. Speak — words appear in real-time on the overlay
  6. Press mouse Back button — overlay closes, text is pasted
  7. Open ~/.whisper_log/transcriptions.html — your recording appears with per-stage status badges

Audio Device Reference

Understanding how different modes access your microphone:

Mode API Device Name Device ID Format
Streaming SDL2 Usb Audio Device #1 Auto (SDL2)
Batch AVFoundation Usb Audio Device #3 WAV 16kHz mono
Device ID Differences: SDL2 and AVFoundation use DIFFERENT device IDs for the same physical microphone. The self-healing mic check in init.lua auto-corrects if the device index shifts after USB reconnect. Override the priority list with the WHISPER_MIC_PRIORITY env var (comma-separated names); the default is Usb Audio Device, MacBook Pro Microphone.
