Installation Guide

Complete step-by-step setup for AI agent replication. Follow every step in order.

Prerequisites

  • macOS (tested on macOS with Apple Silicon — M4 Pro)
  • Homebrew package manager
  • Git
  • A USB microphone (or built-in MacBook mic)
  • Logitech MX Anywhere 3S mouse (optional, for streaming mode mouse buttons)
  • GitHub Copilot subscription (optional, for grammar correction in streaming mode)

Step 1: Install Homebrew Packages

Install the core Whisper and audio recording tools:

brew install whisper-cpp ffmpeg
  • This installs: whisper-cli (batch transcription), whisper-stream (live streaming), whisper-server, and ffmpeg (audio recording)
  • Verify: which whisper-cli → /opt/homebrew/bin/whisper-cli
  • Verify: which whisper-stream → /opt/homebrew/bin/whisper-stream

Step 2: Download Whisper Models

Download both models to support both streaming and batch modes:

mkdir -p ~/whisper-models
# Base model for streaming (141MB, fast ~55ms per step)
curl -L -o ~/whisper-models/ggml-base.en.bin https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-base.en.bin
# Medium model for batch (1.5GB, more accurate)
curl -L -o ~/whisper-models/ggml-medium.en.bin https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-medium.en.bin
Model Selection: The base.en model is 141MB and perfect for real-time streaming. The medium.en model is 1.5GB and provides higher accuracy for batch transcription.

Step 3: Install Hammerspoon

Hammerspoon provides the automation framework for both streaming and batch modes:

brew install --cask hammerspoon
  • Open Hammerspoon from Applications
  • Grant Accessibility permissions: System Settings → Privacy & Security → Accessibility → enable Hammerspoon
  • Grant Microphone permissions when prompted
  • The hs CLI is at /Applications/Hammerspoon.app/Contents/Frameworks/hs/hs
  • Add require("hs.ipc") to init.lua to enable CLI

Step 4: Clone the Repositories

Get the runtime code and documentation:

# Runtime code (batch mode scripts, menubar, config)
cd ~/
git clone https://github.com/bpleger/Careless-Whisper.git

# Documentation (this site)
git clone <your-docs-repo-url> whisper
Forking: If you fork Careless-Whisper to your own account, set up remotes: origin = your fork (pushable), upstream = bpleger/Careless-Whisper (read-only).

Step 5: Configure Hammerspoon

Set up the Hammerspoon configuration to load all Whisper modules:

  • Create/edit ~/.hammerspoon/init.lua — this file loads all modules
  • Copy streaming files to ~/.hammerspoon/:
    • whisper_overlay.lua — streaming overlay UI
    • whisper_streaming.lua — streaming engine
    • whisper_mouse.lua — mouse button bindings
  • Reload Hammerspoon after changes:
killall Hammerspoon; sleep 1; open -a Hammerspoon
⚠️ Warning: NEVER use hs.reload() or the URL scheme to reload Hammerspoon. Always use killall + open. Config changes don't auto-apply.

Step 6: Configure Batch Mode

Configure the batch transcription settings:

  • Edit ~/whisper-mac/upstream/Careless-Whisper/whisper-stt.conf:
    • WHISPER_AUDIO_DEVICE=3 (AVFoundation device ID for your mic — run ffmpeg -f avfoundation -list_devices true -i "" to find yours)
    • WHISPER_LANGUAGE=en
    • WHISPER_MODEL=~/whisper-models/ggml-medium.en.bin
  • Test recording: ffmpeg -f avfoundation -i :3 -ar 16000 -ac 1 -t 5 /tmp/test.wav
  • Test transcription: whisper-cli -m ~/whisper-models/ggml-medium.en.bin -f /tmp/test.wav

Step 7: Configure Mouse Buttons (Optional)

Set up the MX Anywhere 3S side buttons for streaming mode:

  • Install Logi Options+ from Logitech website
  • Open Logi Options+ → select MX Anywhere 3S
  • For Forward button: assign keyboard shortcut Ctrl+Shift+Cmd+9
  • For Back button: assign keyboard shortcut Ctrl+Shift+Cmd+0
Why this works: The MX Anywhere 3S side buttons don't generate standard macOS CGEvents. Hammerspoon cannot detect them directly. Logi Options+ remaps them to keyboard shortcuts that Hammerspoon CAN detect. See Controls Reference for full details.

Step 8: Configure Copilot Auth (Optional)

Enable grammar correction in streaming mode with GitHub Copilot:

mkdir -p ~/.config/careless-whisper
echo '{"token":"ghu_YOUR_COPILOT_TOKEN"}' > ~/.config/careless-whisper/auth.json

Or use the menubar: Hammerspoon menubar → Sign In to Copilot

Optional Feature: Copilot auth is optional. Without it, streaming mode works perfectly — you just don't get automatic grammar correction.

Step 9: Verify Everything

Run through this checklist to confirm your installation is working:

  1. Hammerspoon shows in menubar (○ icon)
  2. Press Ctrl+Cmd+W — icon changes to ● with timer
  3. Speak something, press Ctrl+Cmd+Q — text appears in active app
  4. Press mouse Forward button — overlay appears at bottom of screen with red STREAMING badge
  5. Speak — words appear in real-time on the overlay
  6. Press mouse Back button — overlay closes, text is pasted

Audio Device Reference

Understanding how different modes access your microphone:

Mode API Device Name Device ID Format
Streaming SDL2 Usb Audio Device #1 Auto (SDL2)
Batch AVFoundation Usb Audio Device #3 WAV 16kHz mono
Device ID Differences: SDL2 and AVFoundation use DIFFERENT device IDs for the same physical microphone. The self-healing mic check in init.lua auto-corrects if the device index shifts after USB reconnect. Priority: Usb Audio Device > Sennheiser MB Pro 1 > MacBook Pro Microphone.
