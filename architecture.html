Architecture

System design, data flows, and component breakdown for both batch and streaming modes.

System Overview

The system has two independent transcription pipelines sharing a common Hammerspoon framework:

  • Batch mode: keyboard-triggered, high-accuracy, process-and-paste
  • Streaming mode: mouse-triggered, real-time, live overlay with corrections

Batch Mode Flow

┌─────────────────────┐ ┌──────────────────┐ ┌─────────────────┐ │ Hammerspoon Hotkey │ ──► │ whisper.sh │ ──► │ Clipboard │ │ Ctrl+Cmd+W (start) │ │ - ffmpeg record │ │ (auto-paste) │ │ Ctrl+Cmd+Q (stop) │ │ - whisper-cli │ │ │ └─────────────────────┘ └──────────────────┘ └─────────────────┘

The batch mode flow operates as follows:

  1. User presses Ctrl+Cmd+W → Hammerspoon starts ffmpeg recording (fast-start from Lua, no shell overhead)
  2. Menubar updates to ● with running timer showing recording duration
  3. User presses Ctrl+Cmd+Q → Hammerspoon stops ffmpeg, launches whisper.sh
  4. whisper.sh runs whisper-cli with medium.en model on the recorded WAV file
  5. Applies transcription_corrections.tsv substitutions (longest match first) to fix common errors
  6. Copies result to clipboard and simulates Cmd+V paste into active application
  7. Self-healing mic check runs: verifies device is still correct, auto-corrects if needed

Streaming Mode Flow

┌──────────────────────┐ ┌──────────────────┐ ┌─────────────────┐ │ MX Anywhere 3S │ ──► │ whisper-stream │ ──► │ Live Overlay │ │ Forward btn (start) │ │ - SDL2 audio │ │ (hs.webview) │ │ Back btn (stop+paste)│ │ - base.en model │ │ + Copilot fix │ └──────────────────────┘ │ - 500ms updates │ │ + auto-paste │ └──────────────────┘ └─────────────────┘

The streaming mode flow operates as follows:

  1. Mouse forward button → Logi Options+ remaps → Ctrl+Shift+Cmd+9 → Hammerspoon catches it
  2. whisper_streaming.lua launches whisper-stream as background process
  3. whisper_overlay.lua creates dark overlay panel at bottom of screen
  4. whisper-stream outputs ANSI-encoded text every 500ms with incremental updates
  5. Streaming engine parses ANSI codes: \033[2K\r = partial, \n = confirmed
  6. Hallucination filter discards phantom text patterns (e.g., "Thank you for watching")
  7. Confirmed text displayed in white, partial in gray with blinking cursor
  8. On sentence boundary → Copilot API corrects grammar → green flash animation
  9. Mouse back button → stop whisper-stream → copy accumulated text → paste into active application

Self-Healing Microphone

The system includes automatic microphone device management to handle USB reconnections and device changes:

  • After every batch transcription, whisperCheckMicrophone() in init.lua runs
  • It scans AVFoundation devices using ffmpeg -f avfoundation -list_devices true
  • Pattern-based discovery with priority order (substring match, case-sensitive). Default list is editable in init.lua and overridable at runtime via the WHISPER_MIC_PRIORITY env var (comma-separated substrings):
    1. Usb Audio Device (preferred — generic USB mic)
    2. MacBook Pro Microphone (fallback — built-in)
  • If the device index changed (e.g., USB reconnect), it auto-updates whisper-stt.conf
  • Also runs a device watcher that triggers on USB connect/disconnect events
